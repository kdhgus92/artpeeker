# 아이디어와 작업 노트

이 문서는 아직 확정되지 않은 생각을 적어두는 공간입니다. 깔끔하게 정리하지 않아도 됩니다.

## 다음 기능 후보

- 작품 저장 기능
- 작품 목록 화면
- 작품 상세 화면
- 감상 메모 작성
- 컬렉션 분류
- 로그인 후 사용자 프로필 표시
- Kakao 로그인 성공 후 세션 또는 JWT 처리

## 백엔드 후보 작업

- 사용자 모델 설계
- 작품 모델 설계
- 저장한 작품 API
- 인증 후 사용자 식별 방식 결정
- CORS 설정 정리
- 로컬 개발용 user secrets 구성

## 프론트엔드 후보 작업

- 로그인 후 홈 화면 상태 분기
- 빈 상태 화면
- 작품 카드 컴포넌트
- 목록/상세 라우팅
- 모바일 레이아웃 점검
- 접근성 점검

## 결정 기록

- 기본 브랜치는 `main`입니다.
- 프로젝트 지침은 루트 `AGENTS.md`와 `frontend/AGENTS.md`에 정리합니다.
- 프론트엔드의 현재 톤은 절제된 GitHub 스타일을 유지합니다.

## 열린 질문

- 저장할 작품 데이터는 외부 API에서 가져올지, 사용자가 직접 입력할지?
- 데이터베이스는 무엇을 사용할지?
- Kakao 로그인 후 인증 상태는 쿠키 세션으로 관리할지, 토큰으로 관리할지?
- 배포는 어디에 할지?
