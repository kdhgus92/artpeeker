이 문서는 아직 확정되지 않은 생각을 적어두는 공간입니다. 깔끔하게 정리하지 않아도 됩니다.

## 다음 기능 후보

- 작품 저장 기능
- 작품 목록 화면
- 작품 상세 화면

## 다음 개발 추천

우선순위는 "로그인 이후 사용자별 공간"을 작게 만드는 것입니다.

1. Kakao 로그인 흐름을 실제 앱 로그인으로 완성합니다.
2. 로그인된 사용자 정보를 프론트 상단에 표시합니다.
3. `/gallery` 또는 `/me` 같은 내 공간 페이지를 추가합니다.
4. 내 저장 작품 페이지의 빈 상태 UI를 만듭니다.
5. 이후 작품 저장 API와 DB 모델을 붙입니다.

이 순서가 좋은 이유는 로그인만 되는 앱에서 사용자별 데이터가 있는 앱으로 넘어가는 가장 작은 세로 조각이기 때문입니다. 작품 등록이나 외부 작품 API를 붙이기 전에도 서비스의 기본 형태를 검증할 수 있습니다.

## 현재 로그인 상태 메모

Kakao OAuth 왕복을 Artpeeker 로그인 세션으로 연결하는 기본 흐름을 추가했습니다.

- 백엔드에는 `/auth/kakao`와 `/auth/kakao/callback`이 있습니다.
- OAuth `state` 쿠키 검증, Kakao 토큰 교환, Kakao 사용자 정보 조회 골격은 있습니다.
- 프론트엔드 `/login` 페이지는 Kakao 로그인 버튼과 성공/실패 query string 표시를 처리합니다.
- 로그인 성공 후 HttpOnly 인증 쿠키를 발급합니다.
- 백엔드에는 `/auth/kakao/me`, `/auth/kakao/logout`이 있습니다.
- 홈 화면은 `/auth/kakao/me`를 호출해 로그인 상태와 로그아웃 버튼을 표시합니다.

아직 남은 작업은 로그인한 사용자를 DB의 `User`와 연결하고, 실제 사용자별 데이터를 저장하는 것입니다.
